# 인화 물질 분류 및 파일 저장 보고서

## 1. 개요

본 보고서는 `Mars_Base_Inventory_List.csv` 파일을 읽어
화성 기지 내 적재 화물 목록을 분석하고,
인화성이 높은 물질을 분류하는 과정을 정리한 문서이다.

또한 보너스 과제로 인화성 기준으로 정렬된 데이터를
이진 파일 형태로 저장하고 다시 읽어들이는 작업을 수행하였다.

---

## 2. 수행 내용

이번 과제에서 수행한 내용은 다음과 같다.
- `Mars_Base_Inventory_List.csv` 파일의 내용 읽기 및 출력
- CSV 데이터를 Python의 리스트 객체로 변환
- `Flammability` 값을 기준으로 내림차순 정렬
- 인화성 지수 0.7 이상 항목만 별도로 추출
- 추출한 결과를 `Mars_Base_Inventory_danger.csv` 파일로 저장
- 보너스 과제로 정렬된 전체 목록을 `Mars_Base_Inventory_List.bin` 파일로 저장
- 저장된 이진 파일을 다시 읽어 화면에 출력

---

## 3. 인화성 물질 분류 결과

CSV 파일의 `Flammability` 열을 기준으로 적재 화물 목록을 정렬하였다.

그 결과, 인화성 지수가 높은 물질이 우선적으로 정렬되었으며,
이 중 `Flammability` 값이 0.7 이상인 항목만 위험 물질로 판단하여
별도의 목록으로 분리하였다.

분리된 위험 물질 목록은 `Mars_Base_Inventory_danger.csv` 파일로 저장하였다.

이 과정을 통해 화성 기지 내에서 우선적으로 격리해야 할 물질을
빠르게 식별할 수 있도록 하였다.

---

## 4. 보너스 과제 수행 내용

보너스 과제에서는 인화성 기준으로 정렬된 전체 목록을
이진 파일 형태로 저장하였다.

정렬된 리스트를 문자열 형태로 구성한 뒤,
이를 `utf-8` 방식으로 인코딩하여
`Mars_Base_Inventory_List.bin` 파일에 저장하였다.

그 후 저장된 `.bin` 파일을 다시 읽어 바이트 데이터를 가져오고,
이를 다시 `utf-8`로 디코딩하여 화면에 출력하였다.

이 과정을 통해 텍스트 데이터를 이진 형태로 저장하고
다시 복원하는 과정을 확인할 수 있었다.

---

## 5. 텍스트 파일과 이진 파일의 차이점 및 장단점

### 5.1 텍스트 파일과 이진 파일의 차이점

텍스트 파일은 사람이 읽을 수 있는 문자 형태로 데이터를 저장하는 파일이다.
대표적으로 `.txt`, `.csv`, `.md` 파일 등이 이에 해당한다.

반면 이진 파일은 데이터를 바이트 단위로 저장하는 파일이다.
이진 파일은 사람이 직접 읽기 어렵고,
프로그램이 정해진 방식으로 해석해야 내용을 확인할 수 있다.

이번 과제에서 사용한 csv 파일은 텍스트 파일이며,
보너스 과제에서 생성한 `bin` 파일은 이진 파일이다.

---

### 5.2 텍스트 파일의 장단점

#### 장점
- 사람이 직접 내용을 읽을 수 있다.
- 수정과 확인이 쉽다.
- 디버깅과 검토가 편리하다.

#### 단점
- 저장 효율이 떨어질 수 있다.
- 복잡한 구조를 표현하는 데 한계가 있다.
- 데이터를 사용할 때 문자열을 다시 숫자로 변환해야 할 수 있다.

---

### 5.3 이진 파일의 장단점

#### 장점
- 데이터를 바이트 단위로 저장할 수 있다.
- 프로그램이 처리하기에 적합한 방식으로 저장 가능하다.
- 다양한 형태의 데이터를 저장하는 데 활용할 수 있다.

#### 단점
- 사람이 직접 내용을 확인하기 어렵다.
- 적절한 해석 방식이 없으면 내용을 복원할 수 없다.
- 구조를 모르면 데이터 활용이 어렵다.

---

## 6. 결론

이번 과제를 통해 화성 기지의 적재 화물 목록에서
인화성이 높은 물질을 식별하고 별도로 저장하는 프로그램을 구현하였다.

또한 보너스 과제를 통해 텍스트 파일과 이진 파일의 차이를 실제로 확인하고,
각 저장 방식의 특징과 장단점을 이해할 수 있었다.

텍스트 파일은 사람이 읽기 쉽고 관리가 편리하며,
이진 파일은 프로그램이 데이터를 저장하고 처리하는 데 유리하다는 점을 확인하였다.
