# 살아난 미션 컴퓨터 구현 보고서

## 1. 개요

본 보고서는 화성 기지의 환경 정보를 주기적으로 수집하고 출력하는
미션 컴퓨터 기능을 구현한 과정을 정리한 문서이다.

이번 과제에서는 더미 센서 객체를 이용하여
화성 기지 내부 및 외부 환경 정보를 받아오고,
이를 미션 컴퓨터 객체에 저장하여 JSON 형태로 화면에 출력하도록 구현하였다.

또한 보너스 과제로
특정 키 입력 시 반복 출력을 중단하는 기능과
5분마다 환경값의 평균을 계산하여 출력하는 기능을 추가하였다.

---

## 2. 수행 내용

이번 과제에서 수행한 내용은 다음과 같다.

- `MissionComputer` 클래스 생성
- `env_values` 사전 객체를 속성으로 추가
- 이전 문제에서 구현한 `DummySensor` 클래스를 `ds`라는 이름으로 인스턴스화
- `get_sensor_data()` 메서드 추가
- 센서값을 받아 `env_values`에 저장
- `env_values` 내용을 JSON 형태로 화면에 출력
- 위 작업을 5초마다 반복
- `MissionComputer` 클래스를 `RunComputer`라는 이름으로 인스턴스화
- `RunComputer.get_sensor_data()` 호출을 통해 지속적으로 환경값 출력

---

## 3. 미션 컴퓨터 환경 정보 구성

`MissionComputer` 클래스의 `env_values`에는 다음과 같은 환경 항목을 포함하였다.

- 화성 기지 내부 온도 (`mars_base_internal_temperature`)
- 화성 기지 외부 온도 (`mars_base_external_temperature`)
- 화성 기지 내부 습도 (`mars_base_internal_humidity`)
- 화성 기지 외부 광량 (`mars_base_external_illuminance`)
- 화성 기지 내부 이산화탄소 농도 (`mars_base_internal_co2`)
- 화성 기지 내부 산소 농도 (`mars_base_internal_oxygen`)

이 값들은 모두 사전 객체 형태로 저장하였으며,
센서값을 키 기준으로 구조적으로 관리할 수 있도록 하였다.

---

## 4. 센서 데이터 수집 및 출력 방식

`DummySensor` 객체는 `set_env()` 메서드를 통해
각 환경값을 랜덤으로 생성하고,
`get_env()` 메서드를 통해 현재 환경 정보를 반환한다.

`MissionComputer`의 `get_sensor_data()` 메서드에서는
다음 순서로 동작하도록 구현하였다.

1. `ds.set_env()`를 호출하여 새로운 센서값 생성
2. `ds.get_env()`를 호출하여 센서값 가져오기
3. 가져온 값을 `MissionComputer.env_values`에 저장
4. `json.dumps()`를 사용하여 JSON 형태로 출력
5. `time.sleep(5)`를 사용하여 5초 대기 후 반복

이를 통해 미션 컴퓨터가 환경 센서 데이터를 지속적으로 수집하고,
사용자가 보기 쉬운 JSON 형식으로 표시할 수 있도록 하였다.

---

## 5. JSON 형태 출력

환경 정보는 JSON 형식으로 출력하도록 구현하였다.

예시 형식은 다음과 같다.

```json
{
    "mars_base_internal_temperature": 27.53,
    "mars_base_external_temperature": 9.81,
    "mars_base_internal_humidity": 56.72,
    "mars_base_external_illuminance": 642.18,
    "mars_base_internal_co2": 0.0734,
    "mars_base_internal_oxygen": 5.81
}
```

이와 같이 출력하면 각 환경 정보가 구조적으로 구분되어 보이므로
일반적인 딕셔너리 출력보다 가독성이 더 좋다.

또한 indent = 4 옵션을 사용하여
들여쓰기를 적용함으로써 사람이 읽기 쉽게 구성하였다.

---

## 6. 보너스 과제 수행 내용

보너스 과제에서는 다음 두 가지 기능을 추가하였다.

- 특정 키 입력 시 반복적으로 출력되던 환경값 출력을 중단
- 5분마다 각 환경값의 평균을 계산하여 별도로 출력

### 6.1 출력 중단 기능
반복적으로 환경 정보를 출력하는 과정에서
사용자가 특정 키를 입력하면 출력이 중단되도록 구현하였다.

키 입력 감지를 위해 `msvcrt`를 사용하였으며,
`q` 키가 입력되면 반복문을 종료하고 다음 문구를 출력하도록 하였다.
```bash
System stopped...
```

이를 통해 사용자가 원하는 시점에
미션 컴퓨터의 지속적인 환경 출력 기능을 중단할 수 있도록 하였다.

---
### 6.2 5분 평균값 계산 기능
환경값은 5초마다 한 번씩 수집되므로,
5분 동안 약 60회의 측정값이 누적된다.

이를 위해 `env_history`라는 리스트를 추가하여
각 시점의 환경값을 사전 형태로 저장하였다.

이후 5분이 지나면
각 항목별 측정값을 모두 더한 뒤 측정 횟수로 나누어 평균을 계산하고,
그 결과를 JSON 형태로 출력하도록 하였다.

예를 들어 내부 온도의 평균은 다음과 같은 방식으로 계산된다.

- 5분 동안 수집된 내부 온도 값들을 모두 더함
- 수집 횟수로 나눔
- 소수점 4자리까지 반올림

이를 통해 단순한 순간 측정값뿐만 아니라
일정 시간 동안의 평균 환경 상태도 함께 확인할 수 있도록 하였다.

---

## 7. copy()를 사용한 이유
센서값을 `env_values` 및 `env_history`에 저장할 때
`copy()`를 사용하였다.

이는 딕셔너리 객체가 참조 형태로 전달되기 때문에,
복사를 하지 않으면 이전 측정값까지 함께 변경될 수 있기 때문이다.

예를 들어 과거 측정값을 `env_history`에 저장할 때
같은 딕셔너리 객체를 계속 참조하면
나중에 값이 바뀌었을 때 과거 기록도 의도하지 않게 바뀔 수 있다.

따라서 현재 시점의 센서값을 독립적으로 보존하기 위해
`copy()`를 사용하여 별도의 딕셔너리로 복사한 뒤 저장하였다.

이 과정을 통해 5분 평균 계산에 필요한
과거 데이터가 정확하게 유지되도록 하였다.

---

## 8. 결론
이번 과제를 통해 화성 기지의 환경 정보를 주기적으로 수집하고 출력하는
미션 컴퓨터 기능을 구현하였다.

더미 센서에서 생성한 데이터를 미션 컴퓨터가 받아와
JSON 형식으로 출력하도록 구성함으로써,
화성 기지의 생존 환경을 지속적으로 확인할 수 있는 기초 기능을 만들 수 있었다.

또한 보너스 과제를 통해
사용자 입력에 따른 출력 중단 기능과
5분 평균값 계산 기능을 추가하여,
미션 컴퓨터가 단순 출력 기능을 넘어
환경 데이터를 지속적으로 관리하고 요약할 수 있도록 확장하였다.
