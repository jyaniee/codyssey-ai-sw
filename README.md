**구현 내용**

- Python을 이용한 로그 파일 분석
- 로그 역순 정렬 출력
- 문제 로그 별도 파일 저장

## 문제 03
화성 기지 적재 화물 목록을 분석하여 인화성이 높은 물질을 분류하는 과제

**구현 내용**

- `Mars_Base_Inventory_List.csv` 파일 읽기 및 출력
- CSV 데이터를 Python 리스트 객체로 변환
- 인화성 지수(`Flammability`) 기준 내림차순 정렬
- 인화성 지수 0.7 이상 항목 별도 추출 및 출력
- 위험 물질 목록을 `Mars_Base_Inventory_danger.csv` 파일로 저장
- 정렬된 목록을 `Mars_Base_Inventory_List.bin` 파일로 이진 저장
- 저장된 이진 파일을 다시 읽어 출력
- 텍스트 파일과 이진 파일의 차이점 및 장단점 보고서 작성

## 문제 06
화성 기지 환경 정보를 확인하기 위한 더미 센서를 구현하고,  
환경 값을 생성·출력·기록하는 과제

**구현 내용**

- `DummySensor` 클래스 생성
- `env_values` 사전 객체를 멤버로 추가
- `set_env()` 메서드를 통해 환경 값을 랜덤으로 생성
- `get_env()` 메서드를 통해 환경 값 반환
- `DummySensor` 클래스를 `ds`라는 이름으로 인스턴스화
- `set_env()`와 `get_env()` 호출 결과 확인
- 전체 코드를 `mars_mission_computer.py` 파일로 저장
- 환경 정보를 날짜와 시간과 함께 로그 파일로 저장하는 기능 추가
- 과제 수행 내용을 보고서로 작성

## 문제 07
더미 센서 데이터를 활용하여 미션 컴퓨터가 환경 정보를 지속적으로 수집하고 출력하는 과제

**구현 내용**

- `MissionComputer` 클래스 생성
- `env_values` 사전 객체를 속성으로 추가
- `DummySensor` 클래스를 `ds`라는 이름으로 인스턴스화
- `get_sensor_data()` 메서드를 통해 센서값을 받아 `env_values`에 저장
- 환경 정보를 JSON 형태로 화면에 출력
- 5초에 한 번씩 환경 정보 출력 반복
- `MissionComputer` 클래스를 `RunComputer`라는 이름으로 인스턴스화
- `RunComputer.get_sensor_data()` 호출을 통해 지속적으로 환경값 출력
- 특정 키 입력 시 반복 출력을 중단하는 기능 추가
- 5분마다 각 환경값의 평균을 계산하여 출력하는 기능 추가
- 과제 수행 내용을 보고서로 작성

## 문제 08
미션 컴퓨터의 시스템 정보와 실시간 부하 상태를 확인하고,  
출력 항목을 설정 파일로 제어할 수 있도록 구현하는 과제

**구현 내용**

- `MissionComputer` 클래스에 `get_mission_computer_info()` 메서드 추가
- 운영체계, 운영체계 버전, CPU 타입, CPU 코어 수, 메모리 크기 확인 기능 구현
- `MissionComputer` 클래스에 `get_mission_computer_load()` 메서드 추가
- CPU 실시간 사용량, 메모리 실시간 사용량 확인 기능 구현
- 시스템 정보와 부하 정보를 JSON 형태로 출력
- `MissionComputer` 클래스를 `runComputer`라는 이름으로 인스턴스화
- `runComputer.get_mission_computer_info()` 및 `runComputer.get_mission_computer_load()` 호출
- 시스템 정보 관련 예외처리 추가
- `setting.txt` 파일을 이용하여 출력 항목을 설정하는 기능 추가
- 과제 수행 내용을 보고서로 작성
