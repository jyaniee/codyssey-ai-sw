# 미션 컴퓨터 환경 센서 구현 보고서

## 1. 개요

본 보고서는 화성 기지의 환경 정보를 확인하기 위한 더미 센서 객체를 구현하고,
센서 값을 생성 및 출력하는 과정을 정리한 문서이다.

이번 과제에서는 `DummySensor` 클래스를 정의하고,
화성 기지 내부 및 외부 환경 값을 사전 객체 형태로 관리하도록 구현하였다.
또한 보너스 과제로 환경 정보를 파일에 로그 형태로 저장하는 기능을 추가하였다.

---

## 2. 수행 내용

이번 과제에서 수행한 내용은 다음과 같다.

- `DummySensor` 클래스 생성
- `env_values` 사전 객체를 멤버로 추가
- `set_env()` 메서드를 통해 각 환경 값을 랜덤으로 생성
- `get_env()` 메서드를 통해 환경 값 반환
- `DummySensor` 클래스를 `ds`라는 이름으로 인스턴스화
- `set_env()`와 `get_env()`를 호출하여 값 확인
- 전체 코드를 `mars_mission_computer.py` 파일에 저장

---

## 3. 환경 정보 구성

`env_values` 사전 객체에는 다음과 같은 항목을 포함하였다.

- 화성 기지 내부 온도 (`mars_base_internal_temperature`)
- 화성 기지 외부 온도 (`mars_base_external_temperature`)
- 화성 기지 내부 습도 (`mars_base_internal_humidity`)
- 화성 기지 외부 광량 (`mars_base_external_illuminance`)
- 화성 기지 내부 이산화탄소 농도 (`mars_base_internal_co2`)
- 화성 기지 내부 산소 농도 (`mars_base_internal_oxygen`)

각 항목은 키와 값의 형태로 저장되며,
센서 이름을 기준으로 환경 정보를 쉽게 관리할 수 있도록 하였다.

---

## 4. 환경 값 생성 방식

`set_env()` 메서드에서는 `random.uniform()`을 사용하여
각 환경 항목의 값을 주어진 범위 내에서 무작위로 생성하였다.

각 항목의 범위는 다음과 같다.

- 내부 온도: 18 ~ 30
- 외부 온도: 0 ~ 21
- 내부 습도: 50 ~ 60
- 외부 광량: 500 ~ 715
- 내부 이산화탄소 농도: 0.02 ~ 0.1
- 내부 산소 농도: 4 ~ 7

생성된 값은 소수점이 포함된 실수 형태로 저장하였으며,
실제 센서값처럼 보이도록 `round()`를 사용하여 자릿수를 조정하였다.

---

## 5. 환경 값 반환 및 확인

`get_env()` 메서드는 현재 `env_values`에 저장된 환경 정보를 반환하도록 구현하였다.

이후 `DummySensor` 클래스를 `ds`라는 이름으로 인스턴스화한 뒤,
다음 순서로 메서드를 호출하여 결과를 확인하였다.

1. `set_env()` 호출
2. `get_env()` 호출
3. 반환된 사전 객체의 내용을 출력

이 과정을 통해 더미 센서 객체가 환경 값을 생성하고,
이를 정상적으로 반환하는 것을 확인하였다.

---

## 6. 보너스 과제 수행 내용

보너스 과제에서는 `get_env()` 메서드에
환경 값을 로그 파일로 저장하는 기능을 추가하였다.

로그에는 다음 정보가 포함되도록 구성하였다.

- 날짜와 시간
- 화성 기지 내부 온도
- 화성 기지 외부 온도
- 화성 기지 내부 습도
- 화성 기지 외부 광량
- 화성 기지 내부 이산화탄소 농도
- 화성 기지 내부 산소 농도

로그 저장 시 한 줄에 모든 내용을 기록하는 방식은 가독성이 떨어질 수 있으므로,
각 항목을 줄바꿈하여 여러 줄로 저장하도록 구성하였다.
또한 각 로그 구간이 쉽게 구분되도록 구분선도 함께 추가하였다.

예시 형식은 다음과 같다.

```text
datetime: 2026-03-31 00:25:14
mars_base_internal_temperature: 24.61
mars_base_external_temperature: 7.82
mars_base_internal_humidity: 55.37
mars_base_external_illuminance: 604.12
mars_base_internal_co2: 0.0834
mars_base_internal_oxygen: 5.72
----------------------------------------
```
---

## 7. 로그 저장 방식

로그 파일 저장에는 append 모드를 사용하였다.

즉, 프로그램이 실행될 때마다 기존 로그를 지우지 않고
새로운 환경 정보를 파일의 마지막에 계속 추가하도록 하였다.

이를 통해 시간에 따라 누적되는 환경 변화 기록을 남길 수 있으며,
센서값이 반복적으로 어떻게 변하는지 확인할 수 있도록 하였다.

또한 파일 처리 과정에서 발생할 수 있는 오류를 고려하여
예외 처리도 함께 작성하였다.

## 8. 결론

이번 과제를 통해 화성 기지의 환경 정보를 관리할 수 있는
더미 센서 클래스를 구현하였다.

사전 객체를 사용하여 센서 데이터를 구조적으로 저장하였고,
랜덤 값을 생성하는 메서드와 이를 반환하는 메서드를 분리하여
기능을 명확하게 구성할 수 있었다.

또한 보너스 과제를 통해 환경 정보를 파일에 기록하는 로그 기능을 추가함으로써,
미션 컴퓨터가 환경 정보를 지속적으로 기록하고 관리할 수 있는 기초 기능을 구현하였다.
