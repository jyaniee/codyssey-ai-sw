# 불안정한 미션 컴퓨터 구현 보고서

## 1. 개요

본 보고서는 미션 컴퓨터의 시스템 정보를 확인하고,
현재 부하 상태를 파악하기 위한 기능을 구현한 과정을 정리한 문서이다.

스토리상 미션 컴퓨터가 중간중간 다운되는 현상이 발생하였고,
이를 해결하기 위해 먼저 현재 컴퓨터의 상태를 확인할 수 있는 코드가 필요했다.
따라서 이번 과제에서는 `MissionComputer` 클래스에
시스템 정보 확인 기능과 실시간 부하 확인 기능을 추가하였다.

또한 보너스 과제로 `setting.txt` 파일을 이용하여
출력할 항목을 직접 설정할 수 있도록 기능을 확장하였다.

---

## 2. 수행 내용

이번 과제에서 수행한 내용은 다음과 같다.

- `MissionComputer` 클래스에 `get_mission_computer_info()` 메서드 추가
- 운영체계, 운영체계 버전, CPU 타입, CPU 코어 수, 메모리 크기 출력
- `MissionComputer` 클래스에 `get_mission_computer_load()` 메서드 추가
- CPU 실시간 사용량, 메모리 실시간 사용량 출력
- 시스템 정보 및 부하 정보를 JSON 형식으로 출력
- `MissionComputer` 클래스를 `runComputer`라는 이름으로 인스턴스화
- `runComputer.get_mission_computer_info()` 호출
- `runComputer.get_mission_computer_load()` 호출
- 보너스 과제로 `setting.txt` 파일을 이용한 출력 항목 설정 기능 추가

---

## 3. 구현 방식

이번 과제에서는 시스템 정보를 가져오는 부분에 한해서
별도의 라이브러리 사용이 허용되었으므로,
시스템 정보와 부하 정보를 확인하기 위해 `psutil` 라이브러리를 사용하였다.

운영체계 정보와 CPU 타입 등은 `platform` 모듈을 활용하였고,
CPU 사용률, 메모리 사용률, 전체 메모리 크기 등은 `psutil`을 통해 가져오도록 구현하였다.

시스템 정보를 출력하는 메서드는 다음 두 가지로 구성하였다.

- `get_mission_computer_info()`
- `get_mission_computer_load()`

각 메서드는 정보를 사전 객체 형태로 구성한 뒤,
JSON 형식으로 출력하도록 하였다.

또한 시스템 정보를 가져오는 부분은 예외처리가 되어 있어야 한다는 조건에 맞추어
각 메서드 내부에 `try-except`를 작성하였다.

---

## 4. 시스템 정보 및 부하 출력 기능

### 4.1 시스템 정보 출력 기능

`get_mission_computer_info()` 메서드에서는 다음 항목을 가져오도록 구현하였다.

- 운영체계
- 운영체계 버전
- CPU 타입
- CPU 코어 수
- 메모리 크기

이 값들은 하나의 사전 객체에 저장한 뒤 JSON 형식으로 출력하였다.

예시 형식은 다음과 같다.

```json
{
    "operating_system": "Windows",
    "operating_system_version": "10.0.26200",
    "cpu_type": "Intel64 Family 6 Model 151 Stepping 5, GenuineIntel",
    "cpu_core_count": 12,
    "memory_size_gb": 31.814
}
```
### 4.2 실시간 부하 출력 기능

`get_mission_computer_load()` 메서드에서는 다음 항목을 가져오도록 구현하였다.

CPU 실시간 사용량
메모리 실시간 사용량

이 결과 또한 JSON 형식으로 출력하였다.

예시 형식은 다음과 같다.
```json
{
    "cpu_usage_percent": 9.0,
    "memory_usage_percent": 44.5
}
```

이 기능을 통해 현재 미션 컴퓨터가 어느 정도의 부하를 받고 있는지
실시간으로 확인할 수 있도록 하였다.

---

## 5. 구현 과정에서 확인한 주요 사항

### 5.1 `cpu_count(logical = True)`
CPU 코어 수를 구할 때는 다음과 같은 코드 형식을 사용하였다.
```python
psutil.cpu_count(logical = True)
```
여기서 logical = True는
논리 코어 수를 기준으로 개수를 가져오겠다는 의미이다.

CPU는 물리 코어 수와 논리 코어 수를 다르게 볼 수 있다.
예를 들어 실제 코어가 8개이고 스레드가 16개인 CPU라면

- 물리 코어 수: 8
- 논리 코어 수: 16

으로 구분할 수 있다.

이번 과제에서는 운영체제가 실제로 활용하는 실행 단위에 가까운
논리 코어 수를 출력하도록 하였다.

### 5.2 메모리 크기를 GB 단위로 변환하는 과정
메모리 크기를 출력할 때는 다음과 같은 형태의 코드를 사용하였다.
```python
'memory_size_gb': round(memory_info.total / (1024 ** 3), 3)
```
이 부분은 처음에는 다소 생소할 수 있었기 때문에
구현 과정에서 그 의미를 정확히 이해할 필요가 있었다.

`round()`는 반올림 함수이다.

예를 들어 `round(3.141592, 3)`은
소수점 아래 3자리까지 남기고 반올림한 값을 반환한다.

즉 위 코드에서 뒤의 `3`은
메모리 크기를 소수점 셋째 자리까지 표시하겠다는 의미이다.

또한 `**`는 거듭제곱 연산자이므로
`1024 ** 3`은 `1024 × 1024 × 1024`와 같은 의미이다.

메모리 크기 정보는 보통 바이트(byte) 단위로 반환되기 때문에,
이를 기가바이트(GB) 단위로 바꾸기 위해
`1024`를 세 번 곱한 값으로 나누는 것이다.

즉 이 식 전체는

- 전체 메모리 크기를 바이트 단위에서 GB 단위로 변환하고
- 소수점 셋째 자리까지 반올림하여 출력하는 코드

라고 정리할 수 있다.


### 5.3 예외 처리
이번 과제의 제약사항에서는
시스템 정보를 가져오는 부분에 반드시 예외처리를 해야 한다고 명시되어 있었다.

이에 따라 `get_mission_computer_info()`와 `get_mission_computer_load()` 내부에
`try-except`를 작성하였다.

이를 통해 다음과 같은 상황이 발생하더라도
프로그램이 바로 종료되지 않고 오류 메시지를 출력할 수 있도록 하였다.

- 라이브러리 호출 중 예기치 않은 오류 발생
- 시스템 정보 읽기 실패
- 실시간 부하 정보 읽기 실패

이 방식은 미션 컴퓨터가 불안정한 상태라는 문제 상황과도 잘 맞으며,
프로그램의 안정성을 높이는 데 도움이 된다.

## 6. 보너스  과제 수행 내용

보너스 과제에서는 `setting.txt` 파일을 이용하여
출력되는 정보의 항목을 설정할 수 있도록 코드를 수정하였다.

예를 들어 `setting.txt` 파일에 다음과 같이 항목명을 저장하면
```
operating_system
cpu_type
memory_size_gb
cpu_usage_percent
```
코드에서는 해당 파일을 읽어서
지정된 키만 골라 출력하도록 하였다.

이를 위해 다음 기능을 추가하였다.
- setting.txt 파일 읽기
- 출력 허용 항목 목록 저장
- 전체 시스템 정보/부하 정보 중 설정된 항목만 필터링
- 필터링된 결과만 JSON 형태로 출력

이 기능을 통해 코드 자체를 수정하지 않고도
필요한 정보만 선택적으로 출력할 수 있도록 하였다.

---

## 7. 보너스 과제를 통해 확인한 점
`setting.txt` 방식은 출력 항목을 외부 파일로 분리하여 관리할 수 있다는 장점이 있다.

즉 프로그램 내부 코드를 직접 수정하지 않고도
사용자가 원하는 항목만 선택적으로 출력할 수 있다.

예를 들어 어떤 상황에서는 전체 시스템 정보가 필요할 수 있고,
다른 상황에서는 CPU 사용량이나 메모리 사용량만 필요할 수도 있다.

이번 보너스 과제를 통해
출력 형식을 코드에 고정하지 않고 외부 설정 파일로 제어하는 방식을 경험할 수 있었다.

---

## 8. 결론
이번 과제를 통해 미션 컴퓨터의 시스템 상태를 점검하기 위한 기능을 구현하였다.

운영체계, CPU, 메모리와 같은 기본 시스템 정보와
CPU 및 메모리의 실시간 사용량을 가져와 JSON 형식으로 출력함으로써,
미션 컴퓨터의 현재 상태를 구조적으로 확인할 수 있도록 하였다.

또한 구현 과정에서
`logical = True`의 의미,
메모리 크기를 GB로 환산하는 방식 등
실제로 사용한 코드의 세부 동작도 함께 이해할 수 있었다.

보너스 과제를 통해서는 `setting.txt` 파일을 이용하여
출력 항목을 설정할 수 있도록 확장함으로써,
코드와 설정을 분리하는 방식을 적용해볼 수 있었다.

이를 통해 미션 컴퓨터가 단순히 정보를 출력하는 수준을 넘어,
상황에 맞게 필요한 정보만 선택적으로 보여줄 수 있도록 개선되었다.
